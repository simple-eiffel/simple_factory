Overview

simple_factory provides battle-tested creational design patterns for Eiffel applications. It includes the Factory Pattern for type-safe object creation, Singleton Pattern for shared instances, Object Pool for resource management, and Lazy Initialization for deferred computation.

All patterns are implemented with full Design by Contract support and void-safety. Part of the Simple Eiffel ecosystem.

Quick Start

Installation

  1. Set the environment variable: 
    export SIMPLE_EIFFEL=/d/prod
  2. Add to your ECF file: 
    <library name="simple_factory" location="$SIMPLE_EIFFEL/simple_factory/simple_factory.ecf"/>

Factory Pattern

local
    factory: SIMPLE_TYPE_FACTORY [MY_OBJECT]
do
    create factory.make
    factory.register_type ("widget", {MY_OBJECT})

    -- Create by registered key
    obj := factory.new_instance_from_string ("widget")
end

Object Pool

local
    pool: SIMPLE_OBJECT_POOL [DB_CONNECTION]
    conn: DB_CONNECTION
do
    create pool.make_bounded (5, 20)  -- 5 initial, 20 max

    conn := pool.acquire
    -- use connection...
    pool.release (conn)  -- Returns to pool, reset
end

Lazy Initialization

local
    config: SIMPLE_ONCE_CELL [CONFIG]
do
    create config.make (agent load_config)

    -- Computed only on first access
    print (config.item.database_url)
end

Patterns

Factory Pattern

Abstract and concrete factories for type-safe object creation with registered types.

Singleton Pattern

Lazy-initialized singletons with proper encapsulation and testability.

Object Pool

Reusable object pooling with bounded/unbounded capacity and automatic reset.

Lazy Initialization

Once cells and keyed caches for deferred, cacheable computations.

API Reference

SIMPLE_TYPE_FACTORY [G]

Concrete factory using TYPE manifests for instantiation.

FeatureDescription
makeCreate factory with empty type table
register_type (key, type)Register TYPE for creation via key
new_instance: GCreate fresh instance
new_instance_from_string (spec): GCreate from registered key
has_type (key): BOOLEANIs type registered?

SIMPLE_OBJECT_POOL [G]

Reusable object pool for expensive-to-create objects.

FeatureDescription
make (capacity)Create unbounded pool
make_bounded (capacity, max)Create pool with max size
acquire: GGet object from pool
release (obj)Return object to pool
release_allReturn all objects
count, available_count, in_use_countPool metrics

SIMPLE_ONCE_CELL [G]

Lazy initialization container for expensive computations.

FeatureDescription
make (factory)Create with factory function
item: GGet value (computes if needed)
is_computed: BOOLEANHas value been computed?
invalidateForce recomputation on next access

SIMPLE_CACHED_VALUE [G, K]

Keyed cache with automatic computation.

FeatureDescription
make (factory)Create with factory function
item (key): GGet cached or compute value
is_cached (key): BOOLEANIs value in cache?
invalidate (key)Remove from cache
invalidate_allClear entire cache

SIMPLE_SINGLETON [G]

Singleton holder with lazy initialization.

FeatureDescription
instance: GThe singleton instance
is_initialized: BOOLEANHas instance been created?
reset_instanceClear singleton (testing only)

SIMPLE_CREATABLE

Interface for factory-creatable objects. Inherit this to use with factories and pools.

FeatureDescription
make_defaultInitialize with defaults (deferred)
is_in_default_state: BOOLEANIs in default state? (deferred)
resetReturn to default state

Design by Contract

All classes use comprehensive contracts for correctness guarantees:

Example: SIMPLE_OBJECT_POOL.acquire

require
can_acquire: can_acquire
ensure
result_exists: Result /= Void
in_use_count_increased: in_use_count = old in_use_count + 1

Example: SIMPLE_OBJECT_POOL.release

require
object_exists: a_object /= Void
was_acquired: is_in_use (a_object)
ensure
in_use_count_decreased: in_use_count = old in_use_count - 1
object_reset: a_object.is_in_default_state
